04 Testing harness design
GCC oracle for the Linux kernel problem
When all 16 agents hit the same kernel bug and overwrote each other's fixes, Carlini built a split-compilation harness: GCC compiles a random subset of kernel files, Claude's compiler handles the rest. Each agent works on different failing files in parallel. Delta debugging for pairs that fail together but pass independently.
Context-window hygiene
· Short stdout — errors go to logfiles
· ERROR reason on same line for grep
· --fast flag = 1–10% deterministic sample per agent, random across VMs
· Pre-computed aggregate stats
· Frequent READMEs + progress files for agent orientation in fresh containers
· CI pipeline enforces no regressions on merge

04 What stays the same
🧠
No orchestrator
Carlini's core insight holds. Agents self-direct to "next most obvious" task. TeamCreate enables coordination without mandating a central brain.
🔀
Specialization mix
12 generalists + 4 specialists is the same ratio. Different agent definitions, same strategy of mostly-fungible workers + focused roles.
🧪
Oracle-based testing
GCC as reference compiler is still the right move. Split-compilation and delta debugging remain the best approach for kernel-scale testing.
